INSTACART_ORDER_ANALYSIS

// Patrones de compra en plataforma de comestibles

Python Pandas EDA Data Wrangling Análisis de Patrones
// ————————————

// EL PROBLEMA

Instacart necesitaba entender cómo, cuándo y qué compraban sus usuarios para optimizar inventario, logística y campañas de marketing. Con millones de transacciones acumuladas, el reto era transformar ese volumen de datos en patrones accionables: ¿qué día dimensionar más entregas? ¿qué productos garantizar en stock? ¿cómo personalizar recomendaciones?

// IMG_01: contexto del negocio
// ————————————

// LOS DATOS

El análisis integró 5 datasets relacionales con más de 4.5 millones de transacciones: pedidos, productos, productos por pedido, pasillos y departamentos. El proceso incluyó limpieza profunda de valores ausentes, detección de duplicados (1,362 productos con mismo nombre pero diferente ID), normalización de tipos de datos y merge de las 5 tablas para construir una vista completa del comportamiento de compra por usuario.

// IMG_02: 5 datasets relacionados
// ————————————

// EL ANÁLISIS

Analicé tres dimensiones clave: temporal (patrones por día y hora), de producto (popularidad y tasa de reorden) y de comportamiento (tamaño de pedido, frecuencia, días entre pedidos). Para cada dimensión calculé distribuciones, identifiqué outliers y construí métricas derivadas como tasa de reorden por producto. El análisis reveló concentración extrema: 20 productos concentran una proporción desproporcionada de todas las transacciones.

// IMG_03: patrones día y hora
// IMG_04: productos top reorden
// ————————————

// LOS HALLAZGOS

El domingo es el día pico con 40% más pedidos que el día valle (jueves). El 75% de pedidos ocurre entre 8 AM y 6 PM con pico a las 10 AM. La tasa de reorden global es 59% — los usuarios de Instacart son altamente leales a sus productos. Bananas lideran con 84.4% de reorden. La categoría Produce domina con 29% de ventas totales, y junto con Dairy Eggs representa el 46% del negocio.
